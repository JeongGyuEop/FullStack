2. 스타일 우선순위
  - 캐스케이딩에서 가장 중요하며, 어떤 스타일을 먼저 적용할 것인지를 결정한는 규칙.
  - 얼마나 중요한가?
    > 웹 브라우저에 내용을 표시할 때는 사용자가 지정한 스타일, 제작자의 스타일,
      웹 브라우저가 기본으로 정해 놓은 스타일 3가지의 스타일을 함께 사용
    > "사용자 스타일 => 제작자 스타일 => 브라우저 기본 스타일"의 순서이다.

  - 적용 범위는 어디까지 인가?
    > 중요도가 같은 스타일인 경우, 스타일 적용 범위에 따라 우선순위 결정
    > 스타일 적용 범위가 좁을수록, 즉 정확히 필요한 요소에만 적용할 스타일일수록
      우선순위가 높아진다.
    > !important를 붙이면 그 스타일은 다른 스타일 보다 우선순위가 높다.
    > "!important => 인라인 스타일 => id 스타일 => 클래스 스타일 => 타입 스타일"의
      순서이다.

  - 소스 코드의 작성 순서는 어떠한가?
    > 소스에서 나중에 작성한 스타일이 먼저 작성한 스타일을 덮어쓴다.
    > 즉 나중에 작성한 스타일이 우선순위가 높다.
