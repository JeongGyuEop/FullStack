

	다형성
		- 자바 객체 지향프로그래밍 특징 중 하나
		- 하나를 가지고 여러 개의 기능을 가질 수 있는 능력
		
	다형성의 종류
		- 생성자 오버로딩, 메소드오버로딩, 업캐스팅
		
	업캐스팅
		- 부모클래스자료형의 참조변수 하나를 만들어서
		  여러 자식객체 메모리의 주소번지들을 다양하게 저장시킬 능력
		  
	업캐스팅 단점
		- 부모 클래스의 참조 변수로 자식 객체를 참조할 때,
		  **부모 클래스에 정의된 멤버(필드, 메서드)**만 사용할 수 있습니다.
		  즉, 자식 클래스에 새롭게 정의된 고유의 멤버에는 접근할 수 없습니다.
		
	업캐스팅 장점
		- 부모클래스의 참조변수 하나만 만들어지기 때문에 변수 선언에 메모리 낭비를 줄일 수
		  있고 코드도 짤게 가독성 있게 작성한다.
		  
	다운캐스팅
		- 업캐스팅의 단점을 해결하기 위한 하나의 방법
		- 자식 클래스 자료형의 참조변수에 부모클래스의 참조변수에 저장된
		  자식 객체 메모리 주소번지를 다시 저장시키는 것.
		- 다운 캐스팅은 업캐스팅 후~~~ 다운 캐스팅 해야한다.
		
	다운캐스팅 장점
		- 생성된 자식객체 메모리 전체 포함된 부모 + 자식 멤버들 모두 사용가능하다.
